# GB/T 机械制图

> 本文件是 [`SKILL.md`](SKILL.md) 的中文维护镜像。英文文件是跨平台运行基线；修改任一版本时应同步另一版本及对应核心参考资料。

创建可读、可验证的制造工程图，而不是装饰性 CAD 插图。把几何、线型层级、视图布局、尺寸和技术说明作为一个整体处理。

## 可移植性约定

- 保持核心流程与 Agent、厂商无关。Codex、Claude Code 或其他兼容 Skill 的运行环境必须采用相同的工程验收标准。
- 以 `SKILL.md`、本中文镜像和 `references/` 为权威说明。当前运行时不使用 `agents/openai.yaml` 等可选元数据时可忽略它。
- 执行前发现当前可用的 CAD 应用、MCP/自动化桥、Shell、解析器和渲染器。把能力级步骤映射到实际暴露的安全工具，不假设固定工具命名空间或操作名。
- references 中的操作名和 JSON 载荷只是接口示例，除非当前桥明确暴露，否则不得编造工具调用，也不得声称示例中的检查已经运行。
- 交付物和验证证据应采用标准 CAD/交换/报告格式，保证其他 Agent 或应用可以重新打开并独立验证。

## 必需工作流

1. 分类任务并只加载需要的参考：工程图始终读取 `references/zh-CN/gbt-drafting.md`；AutoCAD/MCP 执行读取 `references/zh-CN/autocad-mcp-workflow.md`；发动机或多系统装配图读取 `references/zh-CN/complex-assembly-drafting.md`；新设计/大幅改版的消费产品、外壳、器具、插口/线缆/控件、运动件或市电产品读取 `references/zh-CN/consumer-product-concept.md`；二维/三维转换或展示读取 `references/cad-workflows.md`；三维零件/装配体读取 `references/cad-3d-modeling.md`；GD&T/GPS、尺寸链或检验读取 `references/gps-inspection.md`；配置、BOM、修订、MBD/PMI、依赖或发布包读取 `references/product-definition-release.md`；任何物理性能计算或仿真结论读取 `references/engineering-analysis.md`；每次 DRC、DFM、配合、干涉或发布审查读取 `references/zh-CN/drc-review.md`。
2. 新设计或大幅改版的消费产品必须先通过适用的概念、采购件、市电/运动件安全、早期预览和产品设计门，才能进入详细尺寸或生产 CAD。选择或安全架构未解决时，停在明确标为临时的概念包。
3. 检查源 CAD、图纸、截图和用户要求。记录具体缺陷，并区分用户给定值、推导值、假设值和未决值。
4. 保留无关的用户几何。只有在需要大范围修复时才在独立区域创建干净替代方案；未经用户授权不得删除旧几何。
5. 布置视图前确认图幅、比例、单位、投影法、打印样式和交付物。GB/T 机械图只有在用户、批准的项目模板和源图都未指定投影法时，才默认采用第一角画法；必须记录此假设，并保持视图位置、投影符号和标题栏数据一致。
6. 建立语义图层和属性后再画几何。线宽从适用标准系列和图幅/比例中选择，并保持清晰的粗细比。实用回退图层如下：
   - `OUTLINE`：粗实线。
   - `THIN`：细实线。
   - `CENTER`：细点画线。
   - `HIDDEN`：细虚线。
   - `HATCH`：细实线。
   - `DIM`：细实线。
   - `TEXT`：细实线。
7. 采用单色打印层级。图层颜色只能辅助 CAD 编辑，并且配置的打印样式必须产生所需线型、线宽和单色输出。
8. 先布置严格对齐的正投影视图，再在有助于表达内部结构时添加剖视图。
9. 从零件向外标注尺寸。优先采用功能基准，避免封闭尺寸链、重复尺寸、交叉和对隐藏线标注。
10. 只有在用户提供或技术依据充分时，才添加公差、配合、表面结构、材料、工艺说明和标题栏数据。未解决的生产数据标记为 `TBD` 或明确标为假设。
11. 执行所选验证层级和表达检查。Tier 1 检查当前受影响几何/视图并报告延期的发布门；Tier 3 必须同时检查模型空间及最新最终打印/导出图，并在可读缩放和全图缩放下确认后才能声明达到发布准备状态。

## CAD 执行规则

### 执行稳定性门槛

- 所有测试、渲染器、CAD 桥接和子进程都必须有单次操作超时与整轮任务超时，禁止等待无界进程或无界嵌套执行单元。
- 超时必须记录为 `TIMEOUT`/`BLOCKED` 并保留 JSON 证据，不能无限重试；只允许清理测试包装器自己拥有的进程。
- 不要为了恢复 CAD 或测试而重启 Codex/Claude 宿主应用。不得隐式启动、关闭或修改用户正在使用的 AutoCAD；先做只读 `preflight`，实时启动必须显式授权。
- 自动化期间禁止弹出外部查看器或抢占前台；PNG/PDF 写入统一工作目录，只返回路径和校验结果。

- 使用原生 CAD 直线、圆弧、圆、多段线、填充、文字和尺寸实体。
- 优先使用关联尺寸对象，禁止用散乱文字和箭头伪造尺寸。
- 几何和标注必须位于预期空间。使用图纸空间时验证视口比例、注释比例和打印线宽。
- 中心线只均匀伸出轮廓少量距离，使用正确中心线型，禁止连续彩色轴线穿过其他零件。
- GB/T 中文文字使用经过验证、汉字和符号覆盖完整且打印可靠的工程字体。`gbenor`/`gbcbig` 或 TrueType 替代字体只是环境选择，不能仅凭文件名判定合规。
- 直径符号和配合代号应属于尺寸对象或关联要求，不能作为零件内部的无关文字。
- 按当前尺寸标注法要求保持尺寸文字水平，并统一文字高度。
- 需要时保持剖切位置标识与剖视图代号配对。普通端视图不得命名为 `SECTION A-A`。
- 使用与所选图幅相称的标准标题栏，禁止过大的装饰文字。
- 不得仅凭外观推断制造关键值。禁止编造配合、公差、表面结构、材料、热处理或工艺要求。
- 把视图一致性视为几何证明义务。即使图面整洁，只要各视图不能描述同一零件/装配体，就不能判定有效。
- 将几何/运动学合理性与工程性能分开。没有载荷、材料属性、边界条件和验收准则时，不得声称强度、刚度、疲劳、热、流体或安全验证通过。
- 以可编辑参数化/原生 CAD 为事实源。除非没有更强来源，STEP/DXF/PDF、网格、渲染和打印图均视为派生物。
- 使用命名参数、稳定基准/轴线、原生特征和原生阵列。装配体用明确配合接口和自由度构建，不使用无法解释的世界坐标偏移拼装。
- 修改或发布前确定准确的配置、修订、单位、CAD/几何内核/API 版本和外部依赖。尽量使用语义 ID/基准而不是易变的面/边索引，并通过实际差异证明修改作用于正确几何。
- 每次可变 CAD 调用后回读所有新建/修改的语义实体，比较请求与实际的类型、图层/角色、坐标/参数、关联和闭合状态。任何无法解释的不一致都是硬停止：回滚当前阶段，只允许一次有限恢复；随后必须切换到验证后端或报告 `blocked`，禁止继续装饰可疑几何。
- 从 `references/cad-workflows.md` 选择验证层级。局部非发布修改采用变更范围检查；只有发布/导出交付或升级触发器使全局证据失效时，才要求完整发布 DRC、最终打印检查和适用的交换格式重导。工具调用成功或渲染漂亮都不能单独证明设计审查通过。
- AutoCAD/MCP 工作必须先证明就绪，再修改；采用有限恢复、跟踪句柄并优先使用原子事务。发布/导出时还要验证实际纸张/比例和重导 DXF；Tier 1 不强制重做未变化的派生物。IPC 成功、保存成功、实体数量或清单都不能单独作为发布证据。
- 复杂装配体在创建详细实体前必须有参数表、组件树、公共基准、预期连接图和视图映射表。每次只构建和审查一个完整子系统，不得发布未经中间拓扑与视觉检查的大批量坐标盲画结果。
- 新消费产品概念或重大形态变更必须先确定人群、场景、接口/插口数量、用户动作、线缆方向和产品意图；至少比较三种低成本轮廓方案，并取得授权选择后才能详细建模。用户提供的已批准概念只有记录修订/权限后才可跳过比较。
- 涉及市电、危险能量或运动/旋转危险时，内部细化前必须建立初步隔离/接地/防火/运动防护/合规架构。未建立前只能创建外形概念和危险/避让包络，禁止标为可制造、安全、合规或可认证。
- 驱动细节的采购件包络和安装接口必须来自供应商资料或受控实物测量。未验证安装尺寸保持 `TBD`，不得进入制造授权范围。
- 参数骨架完成后、详细特征或标注前立即生成预览，检查比例、布局、人群/动作、插口数量/方向、线缆路线、支撑底面、采购件包络和安全/维修避让；骨架失败时返回概念选择。
- 几何/工程图 DRC 与产品设计评审必须分别给出结论。适用消费产品应以明确证据评审外观意图、人体工学、触及/可达、维修、线缆管理和稳定性；几何合法或渲染精美不等于产品设计通过。
- “概念图/原理图”标签可以减少生产标注、分析和细节要求，但不能豁免基本拓扑、实体归属、共同视图来源、比例真实性或视觉完整性。未解释悬空端点、内部交叉、无归属线和开放材料边界必须为零。
- 缺少原生三维、约束、投影、修剪/合并或可靠预览能力时，绘图前定义二维降级范围并准确命名。不得把分别估算的二维概念图当成经过验证的三维装配体或制造图。
- Agent 生成的制造文件只能标记为 `candidate after human review`，除非外部授权流程记录了批准。不得把缺失证据、过期豁免或不可执行检查转成有利的发布结论。
- 工具无法创建或验证所需线型、符号、字体、尺寸样式或打印行为时，必须说明限制并采用最接近且有记录的回退方式，禁止声称完全合规。

## 修订策略

## 五轮项目硬门槛与证据契约（2026-07-19）

以下规则与英文版 `SKILL.md` 同步，适用于 AutoCAD/MCP 的五轮复杂项目验证：

- **文档身份围栏（P0）**：所有 `create/open/activate/save/export` 和修改请求都必须回读 `doc_id`、绝对路径、活动文档、配置、单位和 revision；修改必须携带 `doc_id + expected_revision`。不一致立即 `E_DOCUMENT_ID_MISMATCH`，禁止静默切换用户文档。
- **事务与副作用回滚（P0）**：registry、原生实体和 B-rep 必须在同一原子事务中同步。任一失败必须回滚实体、特征登记、图层、预览文件和日志指针，并以数量、revision 和输出清单证明回滚，不得只凭异常消息判定成功。
- **AABB 语义约束（P1）**：AABB 只能作为 broad-phase 筛选；结果必须记录容器/被容纳组件、`design_role`、坐标系、公差和 `exact_brep|aabb_screening`。AABB 不能证明真实干涉、接触或壁厚。
- **渲染真实性（P1）**：预览必须记录视觉样式、相机、投影、可见组件、非空比例、裁切和哈希，并回读 `material_render_verified`。线框图或未验证材质不得当作 shaded/material review。
- **比例一致性（P1）**：`FIT/NTS` 与固定比例 `1:1、1:2` 等互斥。使用 FIT 时标题栏只能写 FIT/NTS；声明固定比例时必须用实际纸张、MediaBox 和视口比例验证，否则 `PLOT_SCALE_CONSISTENCY=FAIL`。
- **剖视/爆炸预备几何（P1）**：剖视必须预先登记剖切平面和剖切集合；爆炸图必须预先登记组件变换和偏移量。临时拖动相机或截图不构成 section/exploded 证据。
- **证据文件（P1）**：每轮必须保留 `reports/round.json`，包含文档身份、事务回滚、请求/实际差异、DRC、AABB 语义、渲染真实性、比例、交付哈希和复盘结论。缺失或手工修改证据时，该轮只能标记 `BLOCKED`。

R3 箱体/支架、R4 消费品外壳、R5 运动装配的具体失败阈值以英文版 `Five-Round Hard-Gate Mapping` 和仓库 `docs/CAD-5-ROUND-CAMPAIGN-20260719.md` 为准；几何 PASS、STEP 有效或渲染文件存在都不能替代产品审查与证据门。

对于具有稳定源几何和标识的孤立缺陷，直接修改负责的特征/实体，保留回滚点并运行有针对性的依赖检查。只有在缺陷广泛、拓扑/视图权威性不可靠或修复会跨越多个子系统/视图时，才在独立区域创建干净替代方案。替换旧版本前比较几何、标注和打印输出，只有得到明确许可才能删除旧版本。发生门逃逸、原型失败、工具重复失败或误导性交付后，采用 `consumer-product-concept.md` 的证据/根因/临时处理/永久修复/防复发结构。最终答复应总结已修正缺陷、假设、未决 `TBD`、生成文件、延期发布检查及合规限制。
